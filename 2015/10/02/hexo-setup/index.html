Markdown之旅（二）——静态博客框架的搭建

Hexo的安装与简单使用

前言：上一篇Markdown之旅（一）——编辑器的准备折腾完ST3编辑器，现在要开始记录Hexo的折腾过程。

Node.js的安装

直接在官网下载，各种下一步然后就安装完了。

如下图，将npm添加到系统环境变量Path中，;H:\Program Files\nodejs\node_modules\npm 注意修改路径即可。

Git的安装

与Nodejs安装一样，没什么可以说的，也是官网下载，各种下一步。

Hexo的安装

你的静态博客文件夹 鼠标右键 > Git Bash，使用命令安装 $ npm install -g hexo-cli 即可，很简单吧~

建站

如果你按照我说的进行安装，那你的git bash路径应该就是你的静态博客路径了，如果不是，请自行使用 cd 命令进入博客目录，那么接下来就是初始化了，使用下面2命令即可，Hexo将会在当前目录新建所需的文件。

第二个命令可能需要花费一定的时间，耐心等待安装，完成后博客文件夹会多出一些文件/文件夹。

上图所示的就是我的Hexo静态博客目录，使用 $ hexo server 启动服务器，即可使用浏览器访问 localhost:4000 查看效果了，如果4000端口被占用了，可以使用 -p 或 --port 改变，如 $ hexo server -p 12345

刚开始使用，还没有来得及对主题什么的进行调整，使用的是默认的主题。

写作

折腾了那么久，终于可以开始写作了。

$ hexo new [layout] <title>

您可以在命令中指定文章的布局（layout），默认为 post，可以通过修改 _config.yml 中的 default_layout 参数来指定默认布局。

如上图所示就为该文章的创建命令，至此，可以开始写作之旅了。
